# SOP: AliaBot Phase 5.x PWA 중복 제거 및 클라이언트 청소

본 표준 운영 절차(SOP)는 사용자의 컴퓨터 또는 스마트폰 기기에서 구버전 AliaBot PWA 캐시와 OS 레지스트리가 꼬여 **"App으로 다운받기 / 설치"** 아이콘이 노출되지 않고 **"App에서 열기"**로만 강제 고정될 때 이를 클린 소거하는 방법을 가이드합니다.

---

## 1. 개요 및 사전 수집
* **대상 기기**: Windows 10/11 PC 및 Google Chrome 기반 브라우저 환경
* **소요 시간**: 약 3분
* **목적**: 기기에 잘못 등록된 PWA 인스턴스를 소거하여 주소창의 순수 '다운로드 설치' 유도 팝업을 활성화시킵니다.

---

## 2. 세부 청소 절차 (Step-by-Step)

### Step 1. 열려있는 AliaBot 앱 자체에서 삭제
1. 크롬 주소창 우측 끝의 **"App에서 열기"** 아이콘(또는 바탕화면의 이전 AliaBot 아이콘)을 눌러 독립된 앱 창을 실행합니다.
2. 앱 타이틀 바 우측 끝의 **세 점 메뉴(⋮)**를 클릭합니다.
3. 드롭다운 메뉴 하단부의 **`AliaBot 제거...` (Uninstall AliaBot...)** 메뉴를 클릭합니다.
4. 나타나는 최종 확인 팝업에서 **"Chrome에서도 데이터 삭제"** 체크박스를 반드시 체크하고 **[제거]** 버튼을 클릭합니다.

### Step 2. Windows 설치된 앱 목록에서 잔재 청소
1. Windows 검색창에 **"설치된 앱"**을 검색하거나, **[설정] ➡️ [앱] ➡️ [설치된 앱]** 으로 진입합니다.
2. 상단 검색창에 `AliaBot`을 입력합니다.
3. 중복 노출되거나 지워지지 않은 `AliaBot`이 발견되면, 우측 끝의 세 점 메뉴를 눌러 **[제거(Uninstall)]**를 실행합니다.

### Step 3. 크롬 브라우저 앱 관리자 강제 소거
1. 크롬 브라우저 주소창에 `chrome://apps`를 입력하고 접속합니다.
2. 리스트에 `AliaBot` 아이콘이 여전히 남아 있다면, 아이콘에 마우스 우클릭을 한 후 **"Chrome에서 제거..."**를 클릭하여 수동 삭제합니다.

### Step 4. 사이트 저장소 데이터 강제 청소 및 동기화
1. 크롬 브라우저를 열고 `https://aliabot.vercel.app`에 접속합니다.
2. **F12**를 눌러 개발자 도구를 켜고, 상단 탭에서 **[Application]**을 선택합니다.
3. 좌측 메뉴에서 **[Storage]**를 클릭한 후, 중앙 화면 하단의 **[Clear site data]** 버튼을 클릭하여 로컬 스토리지와 서비스 워커 정보를 깨끗이 비웁니다.
4. 브라우저 창에서 **`Ctrl + Shift + R`** (강제 새로고침)을 실행합니다.
5. 크롬 주소창 우측 끝에 모니터 모양의 **"AliaBot 설치" (다운로드)** 버튼이 정상 활성화되는지 확인합니다.

## 3. 자주 묻는 질문 (FAQ)

| 증상 | 발생 원인 | 해결 방안 |
|---|---|---|
| 제거 후 새로고침했으나 계속 'App에서 열기'가 뜹니다. | 백그라운드에 구버전 서비스워커 인스턴스가 좀비 프로세스로 남아 활성화되어 있는 상태입니다. | 브라우저 탭을 완전히 모두 닫은 뒤, 다시 접속하거나 PC를 재부팅하여 세션을 완전 초기화합니다. |
| 설치 아이콘을 눌렀으나 무반응입니다. | Windows 알림 설정 또는 크롬 권한 팝업 차단에 의해 설치 대화 상자가 가려졌을 수 있습니다. | 주소창 왼쪽의 자물쇠 아이콘을 눌러 팝업 및 리디렉션 권한이 '허용' 상태인지 점검합니다. |
