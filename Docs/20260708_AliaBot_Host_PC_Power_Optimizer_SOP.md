# 📖 SOP: 호스트 PC 24시간 상시 기동을 위한 전원 및 랜카드 최적화 가이드 (VSOP)
## 비개발자 사용자를 위한 데스크탑 서버화 윈도우(Windows) 설정 표준 절차서

본 문서는 AliaBot이 모바일 지시사항을 깃허브를 통해 사무실 컴퓨터(데스크탑 1)로 릴레이하여 백그라운드에서 자율 연산하도록 할 때, 컴퓨터 및 인터넷 통신망이 잠들지(Sleep) 않고 상시 온라인 상태를 유지하도록 보장하는 **윈도우(Windows) 운영체제 설정 표준 절차서**입니다.

---

## 1. ⚙️ 핵심 개념 및 작동 원리 (Terminology & Mechanism)

### ① 전원 관리 옵션 (Power Scheme - 전력 관리 프로필)
* **개념**: 컴퓨터의 에너지 소비 효율을 제어하기 위해 윈도우가 제공하는 시스템 제어 체계입니다.
* **작동 원리**: 기본 설정에서는 일정 시간 사용자가 마우스나 키보드를 움직이지 않으면 CPU 성능을 강제로 낮추고 하드디스크 전원을 끄는 **절전 모드(Sleep Mode)** 상태로 전환됩니다. 러너 서버로 쓸 컴퓨터는 이를 `절전 안 함(상시 작동)`으로 고정해야 합니다.

### ② 네트워크 어댑터 절전 (Network Adapter Power Saving - 통신 카드 대기 절약)
* **개념**: 메인보드에 장착된 유선/무선 인터넷 통신 장치(랜카드)가 대기 상태에서 소모하는 전력을 아끼기 위해 장치의 통신 칩셋 전원을 임의로 차단하는 기능입니다.
* **작동 원리**: 윈도우가 컴퓨터가 유휴 상태(Idle)라고 판단하면 랜카드를 잠재워 버리기 때문에, 외부 깃허브에서 들어오는 호출 신호를 인지하지 못하게 됩니다. 이를 장치 관리자에서 영구 차단해야 합니다.

### ③ 윈도우 서비스 등록 (Windows Service Registration - 백그라운드 상주 서비스화)
* **개념**: 프로그램 창을 모니터 화면에 띄워두지 않아도, 컴퓨터 전원이 켜지는 순간 백그라운드 메모리에 자동으로 상주하여 기동되도록 등록하는 윈도우의 기본 프로세스 제어 방식입니다.

## 2. 🚦 단계별 윈도우 설정 절차 (Step-by-Step OS Optimization)

### 1단계: 윈도우 전원 관리 옵션 해제 (절전 모드 차단)
사무실 컴퓨터가 시간이 지나도 스스로 잠들지 않도록 설정합니다.

1. 키보드의 **`시작(Window)`** 키를 누르고 `전원 및 절전 설정`을 검색하여 실행합니다.
2. **화면 및 절전** 탭으로 이동합니다.
3. 다음 전원 옵션들을 전부 **`안 함`** (또는 `해당 없음`)으로 변경합니다:
   * "전원 사용 시 다음 시간 후 화면 끄기" ➡️ **`안 함`**
   * "전원 사용 시 다음 시간 후 PC를 절전 상태로 만들기" ➡️ **`안 함`**
4. 우측 제어판 메뉴에서 `추가 전원 설정`으로 진입하여 전원 관리 옵션을 **`고성능`** (High Performance)으로 선택합니다.

---

### 2단계: 네트워크 어댑터(랜카드) 절전 기능 강제 해제 (통신 끊김 방지)
랜카드가 컴퓨터 유휴 시 잠드는 현상을 원천 방지합니다.

1. 키보드의 **`Windows 키 + X`**를 동시에 누른 뒤 메뉴에서 **`장치 관리자(Device Manager)`**를 실행합니다.
2. 목록에서 **`네트워크 어댑터(Network adapters)`** 항목을 더블클릭하여 하위 항목들을 확장합니다.
3. 현재 사용 중인 유선 랜카드(예: `Intel Ethernet Connection...` 또는 `Realtek...`) 혹은 무선 랜카드를 더블클릭하여 **속성(Properties)** 창을 엽니다.
4. 상단 탭 중 **`전원 관리(Power Management)`** 탭으로 이동합니다.
5. 아래 옵션의 체크를 **해제(Uncheck)** 합니다:
   * **`[ ] 전원을 절약하기 위해 컴퓨터가 이 장치를 끌 수 있음`** (Allow the computer to turn off this device to save power)
6. **확인**을 클릭하여 설정을 저장하고 장치 관리자를 닫습니다.

---

### 3단계: GitHub Actions Runner의 Windows 서비스(Service) 등록
사무실 컴퓨터가 켜질 때마다 콘솔 창을 매번 띄우지 않아도 백그라운드에서 자동으로 깃허브 통신망과 연결되도록 윈도우 서비스로 등록합니다.

1. GitHub Actions Runner 설치 폴더에서 최초 연결 설정을 완수한 뒤, 관리자 권한으로 **파워쉘(PowerShell)**을 켭니다.
2. 설치된 폴더 내부로 진입하여 아래 서비스를 등록하고 시작하는 명령어를 순차 실행합니다:
   ```powershell
   # 1. 윈도우 백그라운드 서비스로 러너 등록
   .\config.cmd --associate-service

   # 2. 서비스 강제 가동 시작
   .\run.cmd --start-service
   ```
3. 이제 사무실 컴퓨터는 로그인 화면 상태(부팅 직후)에만 머물러도 백그라운드에서 상시 깃허브 지시 대기 상태를 안전하게 갱신하게 됩니다.
