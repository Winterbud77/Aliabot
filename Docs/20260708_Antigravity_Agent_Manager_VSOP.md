# 📖 Antigravity 에이전트 매니저(Agent Manager) 활용 VSOP
## 비개발자 도메인 아키텍트를 위한 에이전트 매니저(Agent Manager)와 메인 에디터(Main Editor) 연동 가이드

본 문서는 Antigravity 개발 환경에서 제공하는 **에이전트 매니저(Agent Manager)**와 **메인 에디터 윈도우(Main Editor Window)**의 구조적 연동 관계를 파악하고, 비개발자 사용자가 대화 세션을 관리하며 발생할 수 있는 주요 시나리오를 단계별 캡처 흐름으로 쉽게 따라 할 수 있도록 설계된 시각적 표준 절차서(VSOP - Visual Standard Operating Procedure)입니다.

---

## 1. ⚙️ 핵심 개념 및 작동 원리 (Terminology & Mechanism)

### ① 에이전트 매니저 (Agent Manager - 관제탑)
* **개념**: 여러 개의 독립된 **대화 세션(Conversation Session)**과 각 세션을 담당하는 AI 에이전트들을 통합 모니터링하는 시각 대시보드(Control Dashboard) 창입니다.
* **작동 원리**: 백그라운드 연구(Deep Research)나 장기 실행 프로젝트 등 멀티태스킹(Multitasking - 다중 작업) 작업을 수행하는 에이전트(Agent A, Agent B 등)의 실시간 진행 현황을 중계하여 한눈에 파악할 수 있도록 해 줍니다.

### ② 메인 에디터 윈도우 (Main Editor Window - 작업장)
* **개념**: 실제 소스 코드가 열려 있고 코딩 수정, 파일 탐색, 터미널 컴파일 등 실질적인 개발 작업이 활성화되는 주 편집 창입니다.
* **작동 원리**: 에이전트 매니저에서 선택된 '특정 에이전트 하나'와 일대일로 동기화(Synchronization)되어, 에이전트가 제안하는 코드를 사용자가 즉각 검토하고 빌드하는 단일 태스크 집중 모드로 기동됩니다.

---

## 2. 🗺️ 에이전트 매니저 구조 및 단축키 도식 (System Diagram)

에이전트 매니저와 메인 에디터 창은 아래와 같이 **연결 스위치**를 통해 상호 동기화됩니다.

```
+-----------------------------------+                 +-----------------------------------+
|      Agent Manager (관제탑)       |                 |     Editor Window (실제 작업장)   |
|                                   |   [Ctrl + E]    |                                   |
|   +---------------------------+   | --------------> |   +---------------------------+   |
|   | Agent A (영농 데이터 분석) |   |  (또는 상단의   |   | [활성 파일]               |   |
|   +---------------------------+   |  Open Editor)   |   | 20260707_VTL.md           |   |
|   | Agent B (알리아봇 6.0 챗) |   |                 |   |                           |   |
|   +---------------------------+   |                 |   | [동기화된 에이전트 B]     |   |
|   | Agent C (캘린더 연동 검증) |   |                 |   | 대화 & 실시간 코드 수정   |   |
|   +---------------------------+   |                 |   +---------------------------+   |
+-----------------------------------+                 +-----------------------------------+
```

* **연결 스위치**: 에이전트 매니저 화면 상단의 `Open Editor` 버튼을 누르거나 키보드 단축키 `Ctrl + E`를 입력하면, 관제탑에서 스캔하고 있던 특정 에이전트가 메인 에디터의 사이드 바(Side bar)로 즉각 바인딩(Binding - 밀착 연결)되며 에디터 화면으로 포커싱(Focusing)이 전환됩니다.

## 3. 🚦 단계별 세션 관리 절차 (Step-by-Step SOP)

### [Scenario A] 대화 세션의 명칭(Session Name)을 깔끔하게 바꾸고 싶을 때

1. **에이전트 매니저 진입**:
   * 메인 에디터 우측 상단의 `Open Agent Manager` 버튼을 클릭하여 관제탑 창을 별도로 띄웁니다.
2. **세션 선택 및 컨텍스트 메뉴 호출**:
   * 좌측의 **Conversation History** 목록에서 수정하고자 하는 세션(예: 첫 질문이 그대로 노출되어 제목이 길어진 세션)으로 마우스 커서를 이동합니다.
   * 세션 이름 우측 끝에 나타나는 **점 세 개(...) 아이콘**을 클릭합니다.
3. **이름 변경 실행**:
   * 메뉴에서 **`Rename`**을 선택하고, 새로운 세션 이름(예: `알리아봇 Phase 6.0: Multi-Project Bridge`)을 입력한 뒤 엔터를 누릅니다.
4. **반영 확인**:
   * 세션명이 깔끔하게 수정되며, 메인 에디터 타이틀바의 이름 영역도 실시간으로 동기화되어 정리된 것을 확인합니다.

---

### [Scenario B] 대화 내용 중 중복되거나 불필요한 부분을 편집/삭제하고 싶을 때

1. **말풍선 단위의 개별 편집 가능 여부 확인**:
   * **⚠️ 중요 안내**: AI 대화 기록(Chat History) 내의 특정 말풍선을 사용자가 임의로 선택하여 일부만 버리거나 수정하는 기능은 프로그램 설계상 **지원되지 않습니다**. (대화 이력은 학습과 인과관계 흐름을 기록하는 불변 데이터로 관리됩니다.)
2. **해결 대안 ① (특정 과거 세션 통째로 지우기)**:
   * 더 이상 참조할 필요가 없어진 구형 대화 세션은 에이전트 매니저 좌측 목록의 세션명 우측에 노출되는 **[휴지통 모양 아이콘 (🗑️)]**을 클릭하여 안전하게 완전히 소거(Delete)할 수 있습니다.
3. **해결 대안 ② (현재 세션의 대화 내역 전체 비우기)**:
   * 이번 Phase 6.0에서 우리가 커스텀 개발하여 탑재한 **`[🗑️ 대화 비우기]`** 버튼을 클릭하여 Firestore 내의 메시지를 일괄 소거하고 깨끗한 백지상태에서 새로운 대화를 이어가실 수 있습니다.

---

### [Scenario C] 에이전트 매니저에서 다시 메인 에디터로 안전하게 복귀할 때

1. **복귀 경로 선택**:
   * 에이전트 매니저 창 상단 중앙에 위치한 **`Open Editor`** 버튼에 마우스 커서를 올리면 `Open in Editor` 툴팁이 표기됩니다.
2. **동작 실행**:
   * `Open Editor` 버튼을 클릭하거나, 키보드 단축키 `Ctrl + E`를 입력합니다.
3. **복귀 완료**:
   * 별도로 떠 있던 에이전트 매니저 창이 닫히거나 뒤로 전환되며, 본래 코드를 편집하던 **Antigravity Main Window(메인 에디터 창)**로 즉각 복귀하여 연속적인 개발을 이어가실 수 있습니다.

---

### [Scenario D] 세션 번호(Phase/Decimal) 세분화 및 롤오버(Rollout) 기준

1. **소수점 세션 규칙 (Decimal Session Rules)**:
   * 메이저 Phase 내부에서 개발을 진행하는 도중 대화 흐름이 매끄럽게 보존되고 있으나, 특정 버그 디버깅이나 중간 단계 징검다리가 필요할 때 세션 이름 뒤에 `Phase 6.11`, `Phase 6.13`, `Phase 6.15` 등의 **홀수 단위 소수점 번호**를 부여하여 에이전트 매니저 내 세션을 분화 관리합니다.
2. **메이저 전환 및 롤오버 조건 (Major Rollover Criteria)**:
   * 대화가 길어져서 **컨텍스트 크기(Context Token Limit)**가 너무 커져 응답 속도가 현저히 느려지거나, 하나의 개발 단계(예: 로컬 PC 러너 인프라 구축)가 완벽히 종결될 때는 대화를 새롭게 리셋하여 징검다리를 놓기 위해 **`Phase 6.2`**와 같이 메이저 소수점 첫째 자리 번호로 새로운 세션을 개설해 롤오버(Rollover)합니다.
   * 메이저 롤오버 시에는 "Conversation Session을 전환하여 이어왔다"는 사실을 첫 머리에 명시하고, `Docs/NextSession_ToDo.md` 백로그를 로드하여 대화 문맥 단절을 차단합니다.
