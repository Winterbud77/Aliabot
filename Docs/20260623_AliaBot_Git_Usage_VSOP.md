> **목적 (Purpose)**:
> 비개발자도 Antigravity IDE의 Git 소스 제어(Source Control) 패널을 활용하여 로컬 파일 변경사항을 안전하게 스테이징하고, 버전을 영구 커밋(Commit)하여 내 PC에 저장한 뒤, 최종적으로 GitHub 원격 저장소에 푸시(Push)하여 이중 안전 클라우드 백업을 완수할 수 있도록 안내하는 단계별 행동 절차서입니다.

## 🛠️ Git 버전 기록 및 원격 백업 5단계 표준 절차 (5-Step Commit & Push Procedure)

### 1단계: Git 소스 제어 패널 열기
1. Antigravity IDE 좌측 가장자리의 액티비티 바(Activity Bar)에서 세 번째 위치한 **가지 모양의 아이콘(Source Control)**을 클릭합니다. (단축키: `Ctrl + Shift + G`)
2. 상단에 `Changes` 및 `Staged Changes` 목록이 정상적으로 활성화되는지 확인합니다.

### 2단계: 변경 파일의 상태 기호 식별
각 파일명 오른쪽에 표시되는 알파벳 기호의 의미를 검토합니다:
* **`U` (Untracked)**: 새로 생성되어 Git이 처음 인지한 파일입니다. (커밋 시 버전 관리가 시작됩니다.)
* **`M` (Modified)**: 기존 파일의 내용이 수정된 상태입니다.
* **`D` (Deleted)**: 로컬에서 파일이 삭제되었음을 나타냅니다. 커밋하면 Git 히스토리에서도 삭제 정보가 기록됩니다.

### 3단계: 모든 변경사항 준비 영역으로 이동 (Stage All)
1. `Changes` 타이틀 영역에 마우스 포인터를 올려놓습니다.
2. 오른쪽에 새로 나타나는 **`+` (플러스/Stage All Changes) 버튼**을 클릭합니다.
   * *개별 파일만 선별해서 올리려면, 개별 파일명 오른쪽에 있는 `+` 버튼을 누르면 됩니다.*
3. **결과 확인**: `Changes`에 있던 모든 파일이 상단의 `Staged Changes` 영역으로 이동하며 상태 표시 기호들이 노란색 `A` 또는 주황색 `M` 등으로 변경됩니다.

### 4단계: 커밋 메시지 작성 및 로컬 버전 기록 (Local Commit)
1. `Staged Changes` 상단에 있는 텍스트 입력 칸(`Message (Ctrl+Enter to commit on "master")`)을 클릭합니다.
2. 오늘 작업한 내용을 핵심 요약하여 영문 또는 한글로 명확히 작성합니다.
   * *예시: `Docs: AliaBot 개발/복원 문서 통합 및 프로젝트 정상 상태 백업`*
3. 입력 칸 바로 아래에 있는 **`✓ Commit`** 버튼을 누릅니다.
4. **결과 확인**: `Staged Changes` 및 `Changes` 영역이 완전히 비워지며 깨끗해집니다. (아직 내 컴퓨터에만 저장이 끝난 상태입니다.)

### 5단계: 원격 저장소 전송 및 최종 클라우드 백업 (Remote Push)
1. 커밋이 완료된 직후, 동일한 패널의 버튼이 파란색(또는 금색) **`Sync Changes`** (또는 Publish Branch) 버튼으로 바뀐 것을 확인합니다.
2. 버튼 옆에 **`1 ↑`** (위쪽 화살표와 숫자 1)이 표시되어 있는지 확인합니다. (GitHub로 전송해야 할 커밋이 1개 대기 중이라는 표시입니다.)
3. 이 **`Sync Changes`** 버튼을 클릭합니다.
4. **완료 검증**: 진행 상태 바가 멈추고 버튼의 화살표와 숫자가 완전히 사라지면 GitHub 클라우드로의 전송이 성공한 것입니다. 이제 인터넷 창에서 GitHub 저장소에 들어가도 수정 파일들이 백업된 것을 확인할 수 있습니다.

## 💡 Git Graph 시각적 계기판 읽는 법 (Reading the Git Graph Dashboard)

패널 하단의 `Graph` 섹션에 노출되는 기호와 상태 정보를 통해 내 코드가 안전하게 연동되었는지 확인합니다:

1. **상단 툴바 기능**:
   * **`Auto`**: 버전 그래프의 실시간 자동 고침 상태를 설정합니다.
   * **`Go to Current history item` (🎯)**: 커밋 목록 최상단의 현재 내 컴퓨터 작업 시점(HEAD)으로 화면을 강제 이동시킵니다.
   * **`Fetch from all remote` (☁️⬇️)**: 내 코드를 덮어쓰지 않고, 원격 서버(GitHub)의 변경사항이 있는지만 가져와서 갱신해 줍니다.
2. **참조 라벨(Ref Tags) 동기화 확인**:
   * 내 로컬 컴퓨터의 최신 버전 지점은 **`master`** 태그(보통 파란색 사각형)로 표기됩니다.
   * 원격 GitHub 서버의 최신 버전 지점은 구름 아이콘이 달린 **`origin/master`** 태그로 표기됩니다.
   * **검증 규칙**: `Sync Changes`가 성공적으로 종료되면, 이 두 개의 태그(`master` 및 `origin/master`)가 최상단의 파란색 동그라미(최신 버전)에 **나란히 겹쳐서 표시**됩니다. 이 겹침 상태를 눈으로 확인하는 것이 **완벽히 연동되었다는 보증 마크**입니다.

## ❓ 자주 묻는 질문 및 대처법 (FAQ)

| 증상 / 의문 | 확인 및 조치 방법 |
| :--- | :--- |
| **원하지 않는 임시 파일이 Changes에 섞여 있어요.** | 해당 파일명 오른쪽에 나타나는 **`U-turn` 화살표 아이콘(Discard Changes)**을 누르면, 파일 수정 내역이 취소되고 마지막 커밋 시점의 깨끗한 상태로 되돌아갑니다. (신규 생성 파일 `U`인 경우 파일이 영구 삭제되므로 주의해야 합니다.) |
| **방금 실행한 커밋 메시지를 고치고 싶어요.** | 텍스트 입력창 오른쪽에 있는 `...` (더보기) 버튼을 클릭한 뒤, **Commit -> Undo Last Commit (마지막 커밋 실행 취소)**을 클릭하면 파일들은 스테이징된 상태 그대로 유지되고 커밋만 취소되어 메시지를 수정할 수 있습니다. |
| **Sync Changes를 누르니 로그인 창이 떠요.** | 내 컴퓨터와 GitHub 계정 간의 최초 연동 시 권한 승인을 요구하는 브라우저 창이 열릴 수 있습니다. **'Sign in with your browser'** 버튼을 클릭하여 GitHub 계정 로그인을 연동해 주시면 연동이 완료되어 자동으로 전송됩니다. |
