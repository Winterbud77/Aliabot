# 📋 SOP: Antigravity 대화 세션 분류 및 전역 설정 관리 절차서

본 표준 운영 절차서(SOP)는 사용자가 Antigravity 에디터를 활용할 때 **원하는 명칭과 일련번호로 대화 세션을 시작하는 절차**와, **글로벌 환경 설정(Global Settings) 및 세션 맥락(Session Context)을 안정적으로 전환하며 사용하는 방법**을 기술합니다.

---

## 1. ⚙️ 사전 설정 및 모드 구분 (Mode Distinction)

업무를 진행할 때 현재 열려 있는 설정 창이 **전역 설정을 다루는 창**인지, **AI 에이전트와 소통하는 작업 공간**인지 명확히 인지해야 합니다.

* **글로벌 설정 (Global Settings)**:
  - **진입 방법**: 대화 패널 상단의 시계 모양(대화 기록) 아이콘 바로 옆에 있는 **`...` (점 세 개)** 아이콘 클릭.
  - **용도**: `Customization` (시스템 프롬프트), `MCP Servers` (외부 도구 MCP 연동), `Expert Mode` (전문가 모드 활성/비활성) 수정 시에만 접근.
  - **주의**: 이 옵션들은 대화를 종료해도 에디터 프로그램 전체에 계속 적용되므로, 평상시에는 켜두지 않는 것이 기본입니다.
* **설정 창 (Gear Icon Settings)**:
  - **진입 방법**: 앱 화면 상단 기어(톱니바퀴) 모양 설정 아이콘 클릭.
  - **테스트 규칙 (지인 공유용)**: 지인 테스트 기간 동안 일반 사용자들은 자신의 API 키를 기재하는 **BYOK (브라우저 직접 호출) 방식을 지양하고, 설정 내 `Gemini API Key` 입력 칸을 비워두는 상태(Empty)로 가동**해야 합니다.
  - **이유**: 설정 창의 키를 비워두어야만 Firebase Cloud Functions 백엔드가 호스트(개발자)의 대표 API 키를 시크릿으로 가져와 대리 호출(Proxy Call)을 정상 실행하기 때문입니다.
* **세션 맥락 (Session Context)**:
  - **진입 방법**: 시계 모양(대화 기록) 목록에서 대화방을 선택하거나 새 대화방(`New Chat`)을 개설하여 대화 패널 전체를 질문/답변 창으로 사용하는 상태.
  - **용도**: 소스 코드 수정, 파일 조회, VTL 문서 작성 등 실질적인 개발 진행 시 활용.

## 2. 📝 일련번호 기반의 대화방 생성 절차 (New Session Workflow)

수동 제목 변경이 비활성화된 최신 환경에서, 내가 원하는 순서와 태그로 대화 세션을 정렬하기 위한 표준 생성 프로세스입니다.

### [1단계] 새 대화방(New Session) 개설
* Antigravity 대화 패널 우측 상단의 **`New Chat`** (플러스 모양 또는 새 대화 아이콘) 버튼을 클릭합니다.

### [2단계] 첫 프롬프트(First Prompt) 작성 (일련번호 강제 주입)
* 대화방이 열린 뒤, **첫 번째 질문을 입력할 때 아래 예시 포맷 중 하나를 채택하여 전송**합니다.
* *문장의 가장 맨 처음에 숫자나 대괄호 분류명을 명확하게 기재해야 요약 봇이 해당 식별자를 생략하지 않습니다.*

| 추천 포맷 | 실제 작성 예시 | 예상 생성 제목 |
| :--- | :--- | :--- |
| **대괄호 번호 조합** | `[Phase 5.5-01] AliaBot Obsidian 동기화 구조를 점검합시다.` | `[Phase 5.5-01] AliaBot Obsidian...` |
| **순차 숫자 접두사** | `03. [AliaBot] Firebase Cloud Functions 빌드 에러 디버깅` | `03. [AliaBot] Firebase Cloud...` |
| **날짜형 코드 조합** | `20260624_01: AliaBot 이메일 템플릿 변경 작업을 요청합니다.` | `20260624_01: AliaBot 이메일...` |

### [3단계] 고정 타이틀 확인 및 유지
* 첫 프롬프트 전송 후 AI가 답변을 시작하면, 대화방 상단 헤더 및 좌측 히스토리 목록의 제목이 내가 지정한 접두사 형태(`[Phase 5.5-01] ...`)로 자동 변경되어 고정되었는지 확인합니다.
* 이후 대화는 제목과 무관하게 본래의 개발 질의로 자연스럽게 이어나가시면 됩니다.

## 3. 🚨 예외 상황 및 트러블슈팅 (Troubleshooting)

### Q1. 첫 문장을 전송했는데 제목에 엉뚱한 요약문이 붙었습니다.
* **원인**: 첫 질문의 문장 길이가 너무 길거나, 일련번호 뒤에 나오는 설명 키워드가 빈약하여 요약 알고리즘이 다른 단어를 핵심어로 오해한 경우입니다.
* **해결책**:
  1. 이 상태에서 제목을 수동으로 변경하는 방법은 없습니다.
  2. 만약 순서 정렬을 위해 타이틀이 매우 중요하다면, 해당 방을 삭제하고 새 대화방을 연 뒤 첫 프롬프트를 **단어 단위로 더 간결하게** 입력하여 다시 시작하십시오. (예: `01. Obsidian Sync`)

### Q2. 윈도우 업데이트나 재부팅 후 대화방 목록 자체가 텅 비어 있습니다.
* **원인**: 클라우드 계정 싱크 일시 오류이거나 로컬 인덱스 파일 정보의 일시적 훼손입니다.
* **해결책**:
  1. 먼저 에디터를 완전히 껐다가 켜서 클라우드 자동 로그인이 정상 처리되는지 확인합니다.
  2. 그래도 보이지 않는 경우, 로컬 디스크(`conversations` 폴더)에 보관된 백업 바이너리(`.pb`)를 이식하는 수동 복원 절차를 수행해야 합니다.
  3. 상세 복원 단계는 [AliaBot 대화 세션 수동 복원 표준 절차서(VSOP)](file:///c:/Users/eugene/Projects/Work01_Anti/Docs/20260621_AliaBot_Convo_Session_Restore_VSOP.md)를 정독하여 순서대로 이행하십시오.
