# 🚀 AliaBot Conductor: Phase 5+ Master Roadmap & Handover

> **문서 목적:** '일정/옵시디언 날짜 폴백 및 다중 뱃지 UI 개선, 동기화 경로 이전' 세션을 성공적으로 종료하고, 새로운 'AliaBot Phase 5.6' 세션으로 완벽한 맥락(Context)을 상속하기 위한 인수인계 및 개발 마스터 로드맵입니다.

---

## 1. 🎯 다음 세션(새 대화창) 핵심 목표: "스프레드시트 뷰 및 이메일 채널 확장"

다음 세션에서는 AliaBot이 수집한 데이터를 효율적으로 모아보고 백업할 수 있는 **스프레드시트 형식(Table View)의 대시보드**와 **이메일 알림**을 연동하는 데 집중합니다.

### ✅ 이번 세션 완료 사항 (Phase 5.4 ~ 5.5 일부)
* **Google Calendar 연동 & 정적 폴백 파싱**: AI 파싱 누락 시 자체 정적 파서(`dateParser.js`)로 목표 날짜/시간을 강제 복원하여 구글 캘린더 이벤트 삽입 완료.
* **Obsidian 내보내기 네이밍 고도화**: 내보낼 때 생성 시점이 아닌, 사용자가 지정한 To Do 목표 날짜 기준으로 파일명을 생성하도록 네이밍 연동.
* **다중 뱃지 UI**: 한 메모가 여러 채널(캘린더, 옵시디언 등)로 복수 전송될 시, 카드 상단에 가로 렌더링으로 뱃지가 동시에 노출되도록 UI 개선.
* **옵시디언 100 Source 미러링 이전**: 미러링 문서 복사본들의 종착지를 `100 Source/AliaBot_Docs`로 물리 폴더 이동 및 파워쉘 스크립트(`sync-to-obsidian.ps1`)와 가이드 문서 갱신 완료.
* **원드라이브 `.gitignore` 수립**: 원드라이브 옵시디언 볼트 루트에 `.gitignore` 파일을 배치하여 갱신 빈도가 높은 설정 파일들의 동기화 잠김을 원천 방지.

---

### 📊 Phase 5.5: 대시보드 스프레드시트 뷰 및 CSV 파일 내보내기 (다음 세션 주 목표)
* **엑셀 표 보기 전환 스위치 (Toggle View)**:
  * 기존의 모바일용 세로 카드 리스트 형식 외에, 전체 메모 데이터를 엑셀처럼 격자 형태로 넓게 모아 볼 수 있는 표 보기(Table View) 모드 전환 스위치를 상단에 추가합니다.
* **CSV 데이터 다운로드 (Export to CSV)**:
  * 프론트엔드에 순수 자바스크립트 CSV 변환 엔진을 구현하여, `📥 Excel 다운로드` 버튼 클릭 시 Firestore 내 전체 누적 메모를 즉각 `.csv` 파일로 로컬 기기에 저장할 수 있게 합니다.

---

### 🗓️ Phase 5.6: 이메일(Mail) 발송 채널 확장 및 백필 모니터링
* **이메일(Mail) 발송 기능 (Blaze 요금제 활용)**:
  * Firebase 종량제 요금제를 기반으로 외부 메일 API를 연동하여, 중요한 SOP 노트나 일일 요약 브리핑을 내보내기 목적지 목록에 '이메일'을 선택하여 특정 지인 또는 본인에게 자동으로 발송하는 기능을 통합합니다.
* **날짜 파싱 로직 검증 및 캘린더 백필 모니터링**:
  * 최근 도입한 정적 날짜 파서(`dateParser.js`)와 AI 백필 간의 우선순위 예외 상황들을 모니터링하고 안정성을 높입니다.
